# Anti-Patterns — What NOT to Do

1. Keep duplicate governance sources when `.clinerules` is canonical.
2. Store memory preferences in YAML under `.agent/memory/` (use markdown only).
3. Keep ghost files without explicit read/write trigger contracts.
4. Use external `file:///` links in workspace docs.
5. Recommend forbidden stack items (forbidden-orm, ESLint, Prettier, nodemailer, framer-motion).
6. Reference deprecated Tailwind config patterns instead of v4 CSS-first `@theme`.
7. Use `npm`/`npx` commands in project instructions; use `pnpm` only.
8. Mix historical “cosmic/constellation” meta-system docs with current executable governance.
9. Duplicate role/mode definitions outside `.clinerules § AGENT ROLES`.
10. Keep stale proposal/workflow stubs that have no active process owner.
