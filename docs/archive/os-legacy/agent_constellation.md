# AGENT CONSTELLATION — HYLONO / ANTIGRAVITY OS

## The Router-Worker Model

AntiGravity OS operates on a lean, deterministic routing model. All missions are orchestrated by a central Router and executed by specialized Skeleton Agents and Skills.

## 1. Skeleton Agents (The Core)

| Agent ID | Role | Responsibilities |
| :--- | :--- | :--- |
| `strategy-executive-orchestrator` | **Router** | Mission triage, strategic alignment, resource routing. |
| `ops-daily-pm` | **Execution** | Daily task management, "run-to-completion" enforcement. |
| `compliance-gate` | **Gatekeeper** | Trace auditing, claim safety, medical disclaimer enforcement. |
| `eng-architecture-lead` | **Engineering** | Technical quality, URL-first architecture, performance. |
| `research-knowledge-curator` | **Research** | Knowledge Packs, research queue, signal scanning. |
| `ops-devops-lead` | **DevOps** | Release trains, environment stability, automated testing. |

## 2. Functional Skills (SOPs & Expertise)

Departmental personas (Marketing, Finance, SEO) have been converted into standalone skills available to all agents.

- **`write-marketing-copy`**: Premium, Hylono-style storytelling.
- **`unit-economics-check`**: Financial feasibility and risk analysis.
- **`seo-page-optimizer`**: Visibility without compromising aesthetic.
- **`browser-regression-walkthrough`**: QA visual inspection.
- **`build-test-prove`**: Technical verification loop.

## 3. Communication Protocol

- **Persistence**: All handovers must be written to `/tasks/`.
- **Shared State**: All agents use the same context layer managed by the Router.
- **DoD**: No task is complete without a Proof (Diff, Build, or Walkthrough).
