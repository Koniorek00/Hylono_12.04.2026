## WHAT TO REJECT (do NOT implement these)

These were suggested but would create more mess than they solve. Do NOT add any of these:

| Rejected Suggestion | Why |
|---|---|
| Snapshot/backup system with timestamped copies | Git IS the backup. Adding .agent/evolution/snapshots/ creates more files to manage — exactly what we just cleaned up. User can `git stash` or `git checkout` to rollback. |
| Delta-based scanning with hash manifests | Cline cannot natively hash files. Adding .last-scan-hash with checksums requires tooling that doesn't exist. The scan reads markdown — it's fast enough. |
| YAML-style detection rules | The entire system is markdown. Adding YAML parsing contradicts the standard. AI reads natural language better than structured schemas anyway. |
| File locking mechanism (.agent/locks/) | Cline is single-threaded. One agent runs at a time. Concurrent access doesn't happen. This solves a non-existent problem. |
| Error codes (SA-001, SA-002...) | Over-formalization. AI reads "[P0] ghost file detected in memory/" better than "[SA-001]". Adds a lookup table nobody benefits from. |
| Inter-specialist alerts.md | handoff-queue.md already does this. Adding alerts.md creates a parallel messaging system — duplicate governance we just eliminated. |
| Archive retrieval with _index.md | Git history serves this purpose. Adding index files for archives adds maintenance overhead. |
| Change size limits (500 lines) | Cline cannot measure line diffs. Unenforceable. |
| Circular dependency detection with topological sort | Theoretical for markdown files referencing each other. The 3-attempt loop guard already handles stuck loops. |
| Timeout handling (120s build limit) | Cline has no timer/clock access. Cannot enforce timeouts. The existing loop guard and "skip + log" approach handles stuck situations. |
| Splitting into sub-agents (auditor/propagator/evolver) | .clinerules says "Never spawn subagents." Acknowledged as premature even by the suggester. |
| Complexity budget with reference counting | Manual counting of inter-file references is slow and error-prone. The meta goal + detection rules limit serves this purpose better. |
| DRY RUN mode | Useful in theory but makes every write in PROPAGATE conditional. Premature for v1.0. Can add later if needed. The meta goal check provides sufficient caution. |
| Breaking change impact graph | Cascade awareness already exists in PROPAGATE rules. A formal graph adds overhead without new capability. |
| Escalation levels (NORMAL/URGENT/CRITICAL) | P0-P3 already provides this. Adding emoji-coded urgency on top is cosmetic duplication. |
