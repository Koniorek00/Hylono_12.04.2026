## 🟡 HIGH — Fix this sprint

### 1. Rental Model Underrepresented on Homepage Hero
**Dimension:** Conversion & Business  
**Impact:** Hylono's key differentiator is rental/access model, but hero CTAs focus on "Start Your Journey" without mentioning rental option. Users may bounce thinking products are purchase-only.  
**Files:** [`components/Home.tsx:69-94`](components/Home.tsx:69)

```
→ frontend-specialist: Add secondary CTA "Rent from $150/mo" near primary button
DO: Add visible rental pricing mention in hero section
DO NOT: Remove or replace the existing primary CTA
```

### 2. Content Review Dates Missing on Medical Content
**Dimension:** Content & Compliance  
**Impact:** Medical/wellness content should display "Last reviewed: [date]" for credibility and regulatory compliance.  
**Files:** [`components/TechDetail.tsx`](components/TechDetail.tsx), [`constants.ts`](constants.ts)

```
→ content-product-writer: Add "Last clinically reviewed: [date]" to product pages
DO: Add review date field to product data structure
DO NOT: Backdate or fabricate review dates
```

### 3. Accessibility Needs Verification
**Dimension:** Accessibility  
**Impact:** Skip links and aria-labels are implemented, but WCAG 2.2 AA compliance is unconfirmed. Many Hylono users are elderly or health-compromised.  
**Files:** All component files

```
→ accessibility-specialist: Run axe DevTools audit on key pages
DO: Test with keyboard navigation and screen reader
DO NOT: Assume compliance without testing
```

### 4. API Routes Need Error Handling Audit
**Dimension:** Technical Health  
**Impact:** API routes exist but error handling patterns need verification. Poor error handling leads to bad user experience and potential security issues.  
**Files:** [`app/api/rental/route.ts`](app/api/rental/route.ts)

```
→ backend-specialist: Audit all API routes for proper error handling
DO: Implement standardized error response format
DO NOT: Expose internal error details to clients
```

### 5. Mobile Navigation Testing
**Dimension:** User Experience  
**Impact:** Mobile menu exists but full-screen overlay may cause scroll lock issues on some devices.  
**Files:** [`components/Layout.tsx:145-150`](components/Layout.tsx:145)

```
→ frontend-specialist: Test mobile menu on iOS Safari, Android Chrome
DO: Verify no scroll lock or z-index issues
DO NOT: Assume desktop behavior matches mobile
```

## 💡 OPPORTUNITIES — Strategic enhancements

### 11. Trust Section Expansion
The trust indicators on homepage (FDA Cleared, 5-Year Warranty, 2,500+ Users) could be expanded into a dedicated trust section with:
- Verification links to FDA database
- Warranty details modal
- Customer testimonials with real names (when available)

### 12. Protocol Quick-Start
The ZoneBuilder component is sophisticated but could benefit from a "quick start" questionnaire that recommends protocols based on user goals, similar to the rental checkout's goal selection.

### 13. Partner Locator Promotion
[`components/PartnerLocator.tsx`](components/PartnerLocator.tsx) exists but could be promoted more prominently for B2B conversion. Consider adding "Find a clinic near you" CTA on product pages.

### 14. Product Comparison Feature
Users researching multiple modalities would benefit from a comparison table showing:
- Price comparison
- Use case recommendations
- Synergy combinations
